## Картинка №1. Основная стратегия пробоев с ретестом

### 1. Концепция стратегии

Видимая цепочка процесса:

1. Поиск уровня.
2. Подготовка рынка.
3. Пробой.
4. Подтверждение.
5. Управление позицией.
6. Выход.

Цель блока:

- найти накопление ликвидности;
- войти до или во время пробоя;
- взять импульс после пробоя.

### 2. Поиск уровня

Источники уровней:

- Pivot High;
- Pivot Low;
- Round Number;
- Daily High / Low;
- Cascade Levels.

Критерии:

- минимум 2-3 касания;
- уровень виден на H1/M15;
- не пробивался недавно;
- значимый, с реакцией цены.

### 3. Подготовка пробоя / setup

Факторы подготовки:

- консолидация: диапазон сжимается, ATR падает;
- медленный подход: цена плавно подходит, нет резких откатов;
- тренд: ADX > 25, EMA50 > EMA200.

### 4. Вход — 3 способа

Видимые доли входа:

| Способ | Доля | Контекст |
|---|---:|---|
| Вход заранее | 30% | До пробоя |
| На уровне | 30% | При касании |
| После пробоя | 40% | После закрытия выше уровня |

### 5. Подтверждение пробоя

Видимые условия:

- закрытие выше уровня;
- рост объёма;
- новый локальный максимум.

Пример score:

| Фактор | Баллы |
|---|---:|
| Консолидация | 20 |
| Медленный подход | 20 |
| Тренд | 20 |
| Объём / Активность | 20 |
| Плотность / Поддержка | 20 |

Видимое правило: `Score > 70` → продолжаем удержание.

### 6. Ложный пробой

Видимые условия ложного пробоя:

- пробой уровня;
- возврат под уровень;
- слом структуры / Lower High.

Видимое действие: закрыть лонг → открыть шорт.

### 7. Ретест после пробоя

Видимые этапы схемы:

1. Пробой.
2. Ретест.
3. Удержание уровня.
4. Продолжение движения.

Условия успешного ретеста:

- цена вернулась к уровню;
- не пробила его вниз для long;
- появились покупки / объём;
- новый импульс вверх.

Что делать:

- добавление позиции на ретесте;
- stop-loss под уровень ретеста;
- stop-loss за уровнем ретеста;
- цель: следующий уровень / часть фиксации.

### 8. Управление позицией / добавление

Добавляемся:

- при пробое нового high;
- при пробое локального уровня;
- при пробое каскадного уровня.

Размер добавления:

- подбирать части объёма так, чтобы средняя не портилась;
- обычно +10% - +20% от позиции;
- не более 2 добавлений.

Видимое правило на схеме: скидывать добавленную часть при откате до уровня добавления.

### 9. Выход / take profit framework

Видимая схема фиксации:

| Этап | Доля |
|---|---:|
| Первая фиксация | 30% |
| Вторая фиксация | 50% |
| Остаток / трейлер | 20% |

Видимое правило: stop-loss переносим в безубыток после первой фиксации.

### 10. Работа с плотностью

Основные понятия:

- плотность — большие лимитные заявки;
- поддержка в сторону пробоя;
- разъедание плотности;
- стакан / лента / активность.

Роль плотности:

- может удерживать цену;
- может быть поддержкой;
- при разъедании — пробой.

Использование:

- как точка входа на разъедание;
- как поддержка для отскока;
- как stop-loss, если поглощена.

### 11. Направление рынка и «поводыри»

Видимое правило: истинный пробой гораздо более вероятен по направлению рынка.

Фильтры направления:

- EMA50 > EMA200 → long priority;
- EMA50 < EMA200 → short priority;
- ADX > 25.

Следить за поводырями:

- DXY / доллар;
- US10Y / доходность облигаций;
- Silver / Gold;
- SP500 / Nasdaq.

Видимое предупреждение: если поводырь резко идёт против пробоя — высокая вероятность отскока.

### 12. Типы пробоев

Видимые типы:

- консолидации;
- каскадных уровней;
- локального максимума;
- наклонной / трендовой;
- плотности в сторону пробоя.

## Картинка №2. Дополнительные блоки для реализации

### 1. State machine / конечный автомат

Видимый основной поток:

1. Поиск уровня.
2. Подготовка / Setup.
3. Выбор сценария пробоя.
4. Выбор типа входа.
5. Подтверждение.
6. Ретест, если есть.
7. Добавление позиции.
8. Выход / фиксация.
9. Завершение сделки.

Видимый альтернативный поток включает:

- ложный пробой;
- отказ от сделки.

### 2. Выбор сценария пробоя

Видимый алгоритм вопросов:

- уровень найден?
- есть консолидация перед уровнем?
- есть каскад уровней?
- есть локальный максимум рядом с уровнем?
- есть наклонная / трендовая линия с 3+ касаниями?
- есть плотность в сторону пробоя / поддержка?

Видимые сценарии:

| Scenario | Meaning |
|---|---|
| Сценарий A | Пробой консолидации |
| Сценарий B | Пробой каскадных уровней |
| Сценарий C | Пробой локального максимума |
| Сценарий D | Пробой наклонной |
| Сценарий E | Плотность в сторону пробоя |

Если условия не выполнены: нет сделки.

### 3. Оценка качества пробоя / breakout score

Видимая таблица факторов:

| Фактор | Описание | Вес |
|---|---|---:|
| Консолидация | Диапазон сжимается, ATR падает | 20 |
| Медленный подход | Низкая скорость подхода, нет резких откатов | 20 |
| Тренд | EMA50/EMA200 в сторону пробоя, ADX > 25 | 20 |
| Объём / Активность | Рост объёма при пробое, активное поглощение | 20 |
| Плотность / Поддержка | Плотность в сторону пробоя или разъедание | 20 |

Итого максимальный score: 100.

Правила принятия решения:

- `Score >= 70` → принимаем пробой;
- `Score 50-69` → возможен пробой, сниженный риск;
- `Score < 50` → пропускаем / ждём лучший сетап.

### 4. Risk manager / управление риском

Видимые правила:

- риск на сделку: 0.5% - 1% от депозита;
- размер позиции рассчитывается через risk / stop-loss / стоимость пункта;
- stop-loss за плотностью / за уровнем / по структуре;
- максимум добавлений: не более 2 добавлений;
- максимальный риск по сделке: не более 2R с учётом всех добавлений;
- дневной лимит убытка: не более 3% - 5% от депозита;
- фильтр волатильности: торгуем только при ATR в пределах нормы;
- время торговли: активные сессии, по инструменту.

Видимый пример расчёта:

- депозит: 10 000 $;
- риск на сделку: 1% / 100 $;
- stop-loss: 20 пунктов;
- стоимость пункта: 1 $;
- размер позиции: 5 лотов / mini 5 контрактов.

Контроль риска в сделке:

- перенос в безубыток после первой фиксации;
- следим за структурой;
- выходим при потере импульса;
- не усредняемся против движения.

### 5. Направление рынка и «поводыри»

Фильтр направления:

- EMA50 > EMA200 → long priority;
- EMA50 < EMA200 → short priority;
- ADX > 25.

Поводыри / контекст:

- DXY / доллар;
- US10Y / доходность облигаций;
- Silver / Gold;
- SP500 / Nasdaq.

Правила контекста:

- пробой по направлению поводырей добавляет баллы к score;
- против поводырей вычитает баллы из score;
- резкий разворот поводырей против позиции — повышенный риск ложного пробоя.

Видимое предупреждение: контекст рынка может отменить даже технически идеальный сетап.
