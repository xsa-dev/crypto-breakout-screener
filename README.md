# AYAZSH-PROJECT-TEMPLATE

Шаблон проекта для курса Ayaz Sh по разработке крипто-скринеров, Telegram-ботов и торговых инструментов через AI-агентов.

## Как начать новый проект

1. Открой проект в VSCode.
2. В сообщении агенту опиши, что хочешь собрать: цель проекта, биржу, тип данных, условия сигналов, Telegram-уведомления, торговлю или только наблюдение.
3. Попроси агента сначала изучить `AGENTS.md` и подходящие skills из `.claude/skills/` или `.codex/skills/`.

## Установка и запуск

```bash
uv sync
uv run python -m src.app
```

## Добавление зависимостей

```bash
uv add aiohttp
uv add --dev ruff
```

Не устанавливай библиотеки через `pip install`, если в проекте используется `uv`.

## Проверки качества

Настройки Ruff и Pyright лежат в `pyproject.toml`.

```bash
uv run ruff check .
uv run pyright
```

## Где что лежит

- `AGENTS.md` — правила для AI-агентов и структура проектов курса.
- `CLAUDE.md` — входная инструкция для Claude Code, ссылается на `AGENTS.md`.
- `src/core/` — инфраструктура: конфиг, переменные окружения, логирование.
- `src/database/` — слой БД (SQLite): модели, репозитории.
- `src/app/` — логика приложения: скринер и торговля. Точка входа `src/app/__main__.py`, запуск через `python -m src.app`.
- `src/web/` — веб-админка (FastAPI). ASGI-объект в `src/web/__main__.py`, запуск через `uvicorn src.web.__main__:app`.
- `.claude/skills/` и `.codex/skills/` — 21 project-level skill по крипто-разработке.

## Торговый робот

Робот ловит пампы по фьючерсам Bybit и ставит лимитный BUY-ордер на ретрейс с
TP/SL прямо в ордере (только лонг). Спецификация — в `ТЗ-торгового-робота.md`.

Два независимых процесса, связанные файлом SQLite:

```bash
# Робот (скринер + торговля)
uv run python -m src.app

# Веб-админка (настройки на лету, журнал сигналов и сделок)
uv run uvicorn src.web.__main__:app --host 0.0.0.0 --port 8000
```

Перед запуском скопируй `.env.example` в `.env` и заполни ключи Bybit и Telegram.
Без ключей робот работает «всухую»: скринер ищет сигналы и пишет журнал, но
реальные ордера не выставляет. Торговля включается флагом `trading_enabled` в
админке (по умолчанию выключена).
